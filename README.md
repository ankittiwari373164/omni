# Flow Studio — Multi-Client Dashboard

Automates Google Flow video generation for **many clients**. Each client has its own
cookies, business brief, branding (frame + outro), content calendar, and delivery rules.

Pipeline per video:
**Groq prompt → Google Flow (10s, Omni, 9:16) → apply frame + outro → upload to Drive
and/or YouTube → record everything in Supabase → show in the dashboard.**

---

## What's new vs the old single-page version

- **Left sidebar with clients.** Click a client to open its workspace.
- **Per-client config**: name, business details, cookies, frame PNG, outro clip, delivery toggles.
- **Frame + outro compositing** with ffmpeg. The frame's black centre is keyed out so the
  video shows through; the outro clip is appended at the end.
- **Groq** generates the 10-second Omni prompts and the content calendar.
- **Content calendar** per client; generate a video straight from a calendar item.
- **Google Drive + YouTube** upload via the official APIs and one OAuth client (per-client
  account connection). YouTube metadata (title, description, hashtags, tags) is generated by
  Groq and **"not made for kids" is always set**.
- **Supabase** stores clients, calendars and videos. Every client's videos appear in its
  Videos tab.

## Prerequisites

- Node.js 18+
- **ffmpeg** and **ffprobe** on your PATH (`ffmpeg -version` should work).
  - macOS: `brew install ffmpeg` · Ubuntu: `sudo apt install ffmpeg` · Windows: download from ffmpeg.org
- A Supabase project, a Groq API key, and a Google OAuth client (details below).

## Install

```bash
npm install
npx playwright install chromium
cp .env.example .env      # then fill in .env
```

## 1) Supabase

1. Create a project at supabase.com.
2. Open **SQL Editor** and run the contents of `supabase-schema.sql`.
3. In **Settings → API**, copy the **Project URL** and the **service_role** key into `.env`
   (`SUPABASE_URL`, `SUPABASE_SERVICE_KEY`). The service role key stays on the server only.

## 2) Groq

Get a key at console.groq.com/keys → put it in `.env` as `GROQ_API_KEY`.
Default model is `llama-3.3-70b-versatile` (override with `GROQ_MODEL`).

## 3) Google (Drive + YouTube)

1. In Google Cloud Console, create/select a project.
2. **Enable APIs**: "Google Drive API" and "YouTube Data API v3".
3. Configure the OAuth consent screen (External is fine for testing; add your account as a
   test user). Scopes used: `drive.file` and `youtube.upload`.
4. **Create OAuth client ID → Web application.** Add the redirect URI:
   `http://localhost:3000/api/oauth/google/callback`
5. Put the client id/secret/redirect into `.env`.

> Each client connects its **own** Google account from the Config tab ("Connect Google").
> Tokens are stored per-client in Supabase.

## Run

```bash
npm start
# open http://localhost:3000
```

## Using the dashboard

1. **Add a client** (+ in the sidebar).
2. **Config tab**
   - Business brief — feeds Groq's prompts and calendar.
   - Upload `cookies.json` (export from Google Flow with the Cookie-Editor extension).
   - Upload a **frame** PNG (9:16; the black centre becomes transparent) and an **outro** mp4.
   - Toggle Drive / YouTube upload, set a Drive folder id and default tags, then **Connect Google**.
   - **Save config.**
3. **Calendar tab** — choose number of days → **Generate calendar**. Each item can get a
   prompt ("Make prompt") or go straight to **Generate video**.
4. **Generate tab** — manual prompt or auto from a calendar item, optional reference image,
   live console, and a preview of the finished (framed) video.
5. **Videos tab** — every video for the client with status and Drive/YouTube/Download links.

## File structure

```
flow-studio/
├── server.js              # Express + WebSocket API + pipeline orchestration
├── lib/
│   ├── supabase.js        # Supabase client
│   ├── groq.js            # prompt / calendar / YouTube-metadata generation
│   ├── video.js           # ffmpeg: frame overlay (colorkey) + outro concat
│   ├── google.js          # OAuth + Drive upload + YouTube upload
│   └── flow.js            # Playwright Google Flow automation (10s · Omni · 9:16)
├── public/index.html      # dashboard UI (sidebar, config, calendar, generate, videos)
├── assets/                # stored frames & outro clips
├── outputs/               # generated + composited videos (served at /outputs)
├── uploads/               # transient cookies/images/scripts
├── supabase-schema.sql
├── .env.example
└── package.json
```

## Notes & gotchas

- Flow automation runs Chromium **headful** (`headless:false`) because the Flow UI doesn't
  fully render headless. On a server, run under `xvfb-run -a node server.js`.
- Cookies expire — re-upload from the Config tab when generation reports "Cookies expired".
- Drive/YouTube steps are skipped (with a warning in the console) until the client's Google
  account is connected.
- YouTube uploads default to **public**; change `privacyStatus` in `lib/google.js` if you want
  private/unlisted. New API projects may be limited to unlisted until verified by Google.
- If ffmpeg lives somewhere unusual, set `FFMPEG_PATH` / `FFPROBE_PATH` in `.env`.
